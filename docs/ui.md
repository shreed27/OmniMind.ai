# UI

## Principle

Users should never feel like they are chatting with AI.
They should feel like they are observing and managing a living company.

## Key surfaces

- **Mission Console** — input mission, KPIs, budget, constraints
- **Mission Graph** — clickable nodes showing who, why, confidence, evidence, artifacts, time, cost
- **Organization View** — live org chart with departments and agent status
- **Executive Board** — public debate among CEO/CTO/COO/CFO/CMO/Legal/Research/Security
- **Conflict View** — rejected actions, review committees, vote outcomes
- **Reflection Log** — failures, lessons, patterns, strategy changes
- **Time Machine** — branch, replay, diff past missions

## UX rules

- no endless chat logs as the primary interface
- every action shows source agent, time, and evidence
- silent by default; surfaces exceptions and decisions
- compact terminal-style output, not verbose wrappers
